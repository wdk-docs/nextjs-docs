在 React 和 Next.js 中，可以编写几种不同类型的测试，每种测试都有自己的目的和用例。此页面概述了可用于测试应用程序的类型和常用工具。

## 测试类型

- **单元测试**涉及到单独测试单个单元(或代码块)。在 React 中，一个单元可以是单个函数、钩子或组件。
  - **组件测试**是单元测试的一个更集中的版本，其中测试的主要主题是 React 组件。这可能涉及测试组件的渲染方式、它们与道具的交互以及它们对用户事件的响应行为。
  - **集成测试**包括测试多个单元如何协同工作。这可以是组件、钩子和函数的组合。
- **端到端(E2E)测试**涉及在模拟真实用户场景(如浏览器)的环境中测试用户流。这意味着在类似生产的环境中测试特定任务(例如注册流程)。
- **快照测试**包括捕获组件的渲染输出并将其保存到快照文件中。运行测试时，会将组件的当前渲染输出与保存的快照进行比较。快照中的更改用于指示行为中的意外更改。

## 异步服务器组件

由于`异步`服务器组件是 React 生态系统中的新组件，一些工具并不完全支持它们。同时，我们建议对`异步`组件使用**端到端测试**，而不是**单元测试**。
