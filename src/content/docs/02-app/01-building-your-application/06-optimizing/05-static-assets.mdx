Next.js 可以在根目录中名为`public`的文件夹下提供静态文件，如图像。然后，代码可以从基本 URL（`/`）开始引用`public`中的文件。

例如，如果在`public`中添加`me.png`，则以下代码将访问图像：

```jsx filename="Avatar.js"
import Image from "next/image";

export function Avatar() {
  return <Image src="/me.png" alt="me" width="64" height="64" />;
}
```

<PagesOnly>

此文件夹对`robots.txt`、`favicon.ico`、谷歌网站验证和任何其他静态文件（包括`.html`）也很有用。但是，请确保不要有与`pages/`目录中的文件同名的静态文件，因为这会导致错误。[阅读更多](/docs/messages/conflicting-public-file-page).

</PagesOnly>

<AppOnly>

对于静态元数据文件，如`robots.txt`、`favicon.ico`等，应在内部使用[特殊元数据文件](/docs/app/api-reference/file-conventions/metadata)

</AppOnly>

> 很高兴知道:
>
> - 目录必须命名为`public`。名称不能更改，它是唯一用于提供静态资产的目录。
> - 只有在[构建时](/docs/app/api-reference/next-cli#build)处于`public`目录中的资产才会由 Next.js 提供服务。在请求时添加的文件将不可用。我们建议使用像[AWS S3](https://aws.amazon.com/s3/)这样的第三方服务进行持久文件存储。
