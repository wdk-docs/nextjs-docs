Next.js 提供了各种内置优化，旨在提高应用程序的速度和[核心网络简历](https://web.dev/vitals/). 本指南将介绍您可以用来增强用户体验的优化。

## 内置组件

内置组件抽象了实现通用 UI 优化的复杂性。这些组件是:

- **Images**: 基于本机`<img>`元素构建。图像组件通过延迟加载和根据设备大小自动调整图像大小来优化图像性能。
- **Link**: 基于本机`<a>`标记构建。链接组件在后台预取页面，以实现更快、更平滑的页面转换。
- **Scripts**: 基于本机`<script>`标记构建。脚本组件使您能够控制第三方脚本的加载和执行。

## 元数据

元数据有助于搜索引擎更好地了解您的内容（这可以带来更好的 SEO），并允许您自定义内容在社交媒体上的呈现方式，帮助您在各种平台上创造更具吸引力和一致性的用户体验。

<AppOnly>

Next.js 中的元数据 API 允许您修改页面的`<head>`元素。可以通过两种方式配置元数据：

- **基于配置的元数据**: 在`layout.js`或`page.js`文件中导出[静态的`metadata`对象](/docs/app/api-reference/functions/generate-metadata#metadata-object)或动态[`generateMetadata`函数](/docs/app/api-reference/functions/generate-metadata#generatemetadata-function) 。
- **基于文件的元数据**: 将静态或动态生成的特殊文件添加到管段中。

此外，您还可以使用带有[imageResponse](/docs/app/api-reference/functions/image-response) 构造函数的 JSX 和 CSS 创建动态 Open Graph Images。

</AppOnly>

<PagesOnly>

The Head Component in Next.js allows you to modify the `<head>` of a page. Learn more in the [Head Component](/docs/pages/api-reference/components/head) documentation.

</PagesOnly>

## 静态资产

Next.js`/public`文件夹可用于提供静态资产，如图像、字体和其他文件。`/public`中的文件也可以由 CDN 提供商缓存，以便高效地交付。
