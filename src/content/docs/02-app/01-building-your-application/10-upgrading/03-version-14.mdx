## 从 13 升级到 14

要更新到 Next.js 版本 14，请使用首选包管理器运行以下命令：

```bash filename="Terminal"
npm i next@latest react@latest react-dom@latest eslint-config-next@latest
```

```bash filename="Terminal"
yarn add next@latest react@latest react-dom@latest eslint-config-next@latest
```

```bash filename="Terminal"
pnpm up next react react-dom eslint-config-next --latest
```

```bash filename="Terminal"
bun add next@latest react@latest react-dom@latest eslint-config-next@latest
```

> **Good to know:** 如果使用 TypeScript，请确保还将`@types/react` 和 `@types/react-dom`升级到其最新版本。

### v14 摘要

- Node.js 的最低版本已从 16.14 提高到 18.17，因为 16.x 已经报废。
- 不赞成使用`next export`命令，而赞成使用`output: 'export'`。 请参阅[文档](https://nextjs.org/docs/app/building-your-application/deploying/static-exports)了解更多信息。
- `ImageResponse`的`next/server`导入已重命名为`next/og`。 有一个[codemod 可用](/docs/app/building-your-application/upgrading/codemods#next-og-import)可以安全地自动重命名您的导入。
- `@next/font`包已完全删除，取而代之的是内置的`next/font`。 有一个[codemod 可用](/docs/app/building-your-application/upgrading/codemods#built-in-next-font)可以安全地自动重命名您的导入。
- `next-swc`的 WASM 目标已被删除。
