本页将指导您如何在 Next.js 应用程序中定义和组织路由。

## 创建路由

Next.js 使用基于文件系统的路由器，其中**folders**用于定义路由。

每个文件夹代表一个[**路由**段](/docs/app/building-your-application/routing#route-segments)，映射到**URL**段。要创建[嵌套路由](/docs/app/building-your-application/routing#nested-routes)，可以将文件夹嵌套在彼此内部。

在本例中，`/dashboard/analytics`URL 路径*不*可公开访问，因为它没有相应的`page.js`文件。此文件夹可用于存储组件、样式表、图像或其他共有文件。

> **很高兴知道**: `.js`, `.jsx`, 或 `.tsx` 文件扩展名可以用于特殊文件。

## 创建 UI

[特殊文件约定](/docs/app/building-your-application/routing#file-conventions)用于为每个管段创建 UI。最常见的是[页面](/docs/app/building-your-application/routing/pages-and-layouts#pages)显示路由唯一的 UI，[布局](/docs/app/building-your-application/routing/pages-and-layouts#layouts)显示跨多个路由共享的 UI。

例如，要创建第一个页面，请在`app`目录中添加一个`page.js`文件，然后导出一个 React 组件：

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>;
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>;
}
```
