开始使用 Next.js 最简单的方法是使用`create-next-app`。此 CLI 工具使您能够快速开始构建新的 Next.js 应用程序，并为您设置一切。

您可以使用默认的 Next.js 模板创建一个新的应用程序，也可以使用其中一个[官方的 Next..js 示例](https://github.com/vercel/next.js/tree/canary/examples).

### 交互

您可以通过运行以下命令以交互方式创建新项目：

```bash filename="Terminal"
npx create-next-app@latest
```

```bash filename="Terminal"
yarn create next-app
```

```bash filename="Terminal"
pnpm create next-app
```

```bash filename="Terminal"
bunx create-next-app
```

然后，系统将提示您以下内容:

```txt filename="Terminal"
What is your project named?  my-app
Would you like to use TypeScript?  No / Yes
Would you like to use ESLint?  No / Yes
Would you like to use Tailwind CSS?  No / Yes
Would you like to use `src/` directory?  No / Yes
Would you like to use App Router? (recommended)  No / Yes
Would you like to customize the default import alias (@/*)?  No / Yes
```

一旦您回答了提示，将根据您的回答创建一个具有正确配置的新项目。

### 非交互式

也可以传递命令行参数以非交互方式设置新项目。

此外，您可以通过在默认选项前面加上`--no-`来否定它们 (例如 `--no-eslint`).

查看 `create-next-app --help`:


### 为什么要使用创建 Next 应用程序？

`create-next-app` 允许您在几秒钟内创建一个新的 Next.js 应用程序。它由 Next.js 的创建者正式维护，并包括许多好处:

- **互动体验**: 正在运行`npx create-next-app@latest`(没有任何争论)启动一种互动体验，引导您完成项目设置。
- **零依赖项**: 初始化项目只需一秒钟。创建下一个应用程序没有依赖项。
- **离线支持**: 创建下一个应用程序将自动检测您是否离线，并使用本地包缓存引导您的项目。
- **支持示例**: Create Next App 可以使用 Next.js 示例集合(例如 `npx create-next-app --example api-routes`)或任何公共 GitHub 存储库中的示例引导您的应用程序。
- **已测试**: 该包是 Next.js monorepo 的一部分，并使用与 Next.js 本身相同的集成测试套件进行测试，确保它在每个版本中都能正常工作。
