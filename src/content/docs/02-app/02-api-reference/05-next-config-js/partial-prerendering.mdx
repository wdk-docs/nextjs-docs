---

> **警告**: 部分预渲染是一项实验性功能，目前**不适用于生产环境**。

部分预渲染是一种实验性功能，它允许预渲染路由的静态部分，并从缓存中提供动态漏洞，所有这些都在一个 HTTP 请求中进行。

部分预渲染在中可用`next@canary`:

```bash filename="Terminal"
npm install next@canary
```

可以通过设置实验性的`ppr`标志来启用`部分预渲染`：

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    ppr: true,
  },
};

module.exports = nextConfig;
```

> **很高兴知道:**
>
> - 部分预渲染尚未应用于客户端导航。我们正在积极开展这方面的工作。
> - 部分预渲染仅针对[Node.js 运行时](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)设计。当您可以立即为静态 shell 提供服务时，不需要使用 Node.js 运行时的子集。

在[Next.js 学习课程](/learn/dashboard-app/partial-prerendering)中了解有关部分预渲染的更多信息.
