有些软件包可以导出数百或数千个模块，这可能会导致开发和生产中的性能问题。

将包添加到`experimental.optimizePackageImports`中只会加载您实际使用的模块，同时仍然可以方便地编写带有许多命名导出的导入语句。

```js filename="next.config.js"
module.exports = {
  experimental: {
    optimizePackageImports: ["package-name"],
  },
};
```

像这样的库 `@mui/icons-material`, `@mui/material`, `date-fns`, `lodash`, `lodash-es`, `react-bootstrap`, `@headlessui/react`, `@heroicons/react`, 和 `lucide-react`默认情况下已经优化。
