title: unstable_noStore
description: 函数unstable_noStore的API引用。
---

`unstable_noStore`可用于声明性地选择退出静态呈现，并指示不应缓存特定组件。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function Component() {
  noStore();
  const result = await db.query(...);
  ...
}
```

> **很高兴知道**:
>
> - `unstable_noStore` 在`fetch`上相当于 `cache: 'no-store'`
> - `unstable_noStore` 优先于 `export const dynamic = 'force-dynamic'`因为它更细粒度，可以按组分使用

- 在[`unstable_cache`](/docs/app/api-reference/functions/unstable_cache)内使用 `unstable_noStore`不会选择退出静态生成。相反，它将根据缓存配置来决定是否缓存结果。

## 用法

如果您不希望将其他选项传递给 `fetch`, 喜欢 `cache: 'no-store'` 或 `next: { revalidate: 0 }`, 你可以使用 `noStore()` 作为所有这些用例的替代。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function Component() {
  noStore();
  const result = await db.query(...);
  ...
}
```
